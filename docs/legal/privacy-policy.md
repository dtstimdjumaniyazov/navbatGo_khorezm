# Политика конфиденциальности NavbatGo

> ⚠️ **Черновик для юриста.** Этот документ — рабочая заготовка, составленная
> на основе того, какие данные реально собирает и хранит система NavbatGo
> (см. `core/models.py`). Он не является юридической консультацией и должен
> быть проверен и утверждён квалифицированным юристом перед публикацией,
> с учётом актуальной редакции Закона Республики Узбекистан «О персональных
> данных» (ЗРУ-547) и подзаконных актов. Поля в квадратных скобках `[...]`
> нужно заполнить реальными реквизитами.

## 1. Общие положения

1.1. Настоящая Политика конфиденциальности (далее — «Политика») определяет
порядок обработки персональных данных пользователей сервиса NavbatGo
(Telegram-бот, веб-панель, мобильное приложение) — сервиса онлайн-записи
в автосервисы (далее — «Сервис»).

1.2. Оператором персональных данных является **[наименование юридического
лица / ИП, ПИНФЛ/ИНН, юридический адрес]** (далее — «Оператор»).

1.3. Используя Сервис (регистрируясь через Telegram, оформляя запись через
бота, веб-сайт или мобильное приложение), пользователь подтверждает, что
ознакомился с условиями настоящей Политики и даёт согласие на обработку
своих персональных данных на изложенных ниже условиях.

## 2. Какие данные собираются

В зависимости от роли пользователя (клиент или мастер/сотрудник автосервиса)
Сервис обрабатывает:

| Данные | Кто предоставляет | Зачем |
|---|---|---|
| Telegram ID | автоматически, при входе через Telegram | идентификация пользователя, отправка уведомлений |
| Имя | клиент/мастер | отображение в записи, обращение |
| Номер телефона | клиент/мастер (не обязателен для клиента с Telegram) | связь по записи |
| Данные автомобиля (марка/модель) | клиент, по желанию | подготовка сервиса к визиту |
| История записей (дата, время, услуга, статус) | формируется автоматически | ведение расписания, история обращений |
| Счётчик неявок (`no_show_count`) | формируется автоматически | добросовестность клиента, виден только мастеру сервиса, где были записи |
| Причина отмены записи мастером | вводится мастером | информирование клиента о причине отмены |
| Для мастеров: фото профиля, биография, стаж, язык интерфейса | мастер | публичная витрина автосервиса |

Сервис **не запрашивает и не хранит** платёжные данные (банковские карты и т.п.) — на данный момент оплата через Сервис не производится.

## 3. Цели обработки данных

- организация записи клиента к автосервису и учёт этой записи;
- направление уведомлений о записи (напоминание, отмена, сдвиг очереди) через Telegram;
- предоставление мастеру автосервиса информации, необходимой для оказания услуги;
- статистика качества обслуживания (доля отменённых записей) — доступна только владельцу платформы (суперпользователю), не третьим лицам.

## 4. Правовое основание обработки

Обработка осуществляется на основании согласия пользователя, которое даётся
им при первом входе в Сервис через Telegram. Пользователь вправе отозвать
согласие, обратившись по контактам, указанным в разделе 8 — в этом случае
его аккаунт и связанные данные удаляются, за исключением случаев, когда
их сохранение требуется по закону.

## 5. Кому передаются данные

5.1. Данные клиента (имя, телефон, авто, история записей, счётчик неявок)
доступны **только тому автосервису**, к которому клиент записывался —
мастер одного сервиса не видит клиентов и записи другого сервиса
(техническая изоляция на уровне системы).

5.2. Данные не передаются рекламным сетям, аналитическим сервисам третьих
лиц и не продаются.

5.3. Уведомления доставляются через Telegram Bot API (Telegram FZ-LLC) —
технический канал доставки сообщения, содержание которого относится к
конкретной записи пользователя.

## 6. Где и как хранятся данные

6.1. Персональные данные пользователей — граждан Республики Узбекистан —
хранятся на серверах, расположенных **[указать: на территории Республики
Узбекистан / уточнить у хостинг-провайдера]**.

6.2. Доступ к базе данных ограничен техническим персоналом Оператора.
Пароли пользователей веб-панели хранятся в виде хэшей, доступ к API
защищён токенами (JWT).

6.3. Срок хранения: данные хранятся, пока учётная запись активна. При
удалении аккаунта данные удаляются или обезличиваются в течение
**[указать срок, напр. 30 дней]**, за исключением записей, которые
требуется сохранить для бухгалтерского/налогового учёта автосервиса.

## 7. Права пользователя

Пользователь вправе:
- запросить копию своих данных, хранящихся в Сервисе;
- потребовать исправить неточные данные (например, неверный номер телефона);
- потребовать удаления своих данных и учётной записи;
- отозвать согласие на обработку персональных данных.

Запросы направляются по контактам из раздела 8 и рассматриваются в срок,
установленный законодательством Республики Узбекистан.
