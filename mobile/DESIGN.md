# NavbatGo — мобильное приложение для сервисов (Expo / React Native)

Каркас этапа 2. Дизайн построен на выводах после отказа от прошлой панели:
пользователю были непонятны **сетка расписания** и **логика в целом**.

## Принципы (что изменилось по сравнению с прошлой версией)

1. **Никакой сетки-таймлайна.** Только простой вертикальный список записей
   в хронологическом порядке — как список сообщений.
2. **Первый экран отвечает на вопрос «что мне делать сейчас?»**
   Сверху — машины на постах (и кнопка «Готово»), ниже — кто на очереди
   (и кнопка «Приехал»). Ничего листать и искать не нужно.
3. **Кнопки — глаголами, по одной главной на карточку.**
   «Приехал — начинаю», «Готово», «Не приехал». Без словаря статусов.
4. **Запись с улицы — три шага на одном экране:** услуга → время → имя.
   Свободное время показывается готовыми кнопками, пост выбирается сам.

## Экраны

- **Вход** — одна кнопка «Войти через Telegram»: открывается бот
  @navbatgo_khorezm_bot, мастер жмёт Start, приложение получает токены
  (device-code flow, /api/auth/telegram/start|poll).
- **Сегодня** (главный): секции «На посту сейчас» → «Дальше» → «Завершённые».
  Действия: Готово, +15/+30/+60 (с подтверждением сдвига очереди и
  Telegram-уведомлениями клиентам), Приехал, Не приехал, Отмена.
  Телефон клиента — кликабельный (звонок). Автообновление раз в 60 сек +
  pull-to-refresh.
- **Записать**: список услуг → дата (Сегодня/Завтра/дни недели) и свободные
  слоты кнопками → имя/телефон/машина → «Записать». Дедуп клиентов по
  телефону происходит на сервере.
- **Профиль**: имя, о себе, стаж; о сервисе, Instagram-ссылка,
  «за сколько часов напоминать клиентам» (настройка уведомлений сервисом);
  выход.

## Отложено сознательно (не забыто)

- Фото/галерея профиля из приложения (пока — через веб-панель), пуш-уведомления
  в приложение (уведомления идут в Telegram — так решено), узбекский язык,
  экран «вся неделя», офлайн-режим, нативная сборка в сторы (сначала Expo Go).

## Запуск

```bash
cd mobile
npm start          # затем отсканировать QR в Expo Go
```

API-адрес берётся из `.env` → `EXPO_PUBLIC_API_URL` (по умолчанию —
ngrok-домен API). Бэкенд должен быть запущен и доступен с телефона.
