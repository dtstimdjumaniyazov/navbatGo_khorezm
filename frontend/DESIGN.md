# Дизайн-проект: Рабочая панель автосервиса (Рабочее место мастера)

## 1. Архитектура и навигация приложения
Приложение состоит из одного основного функционального экрана (Одностраничное приложение - SPA), чтобы минимизировать когнитивную нагрузку и количество кликов. Все дополнительные действия выполняются через модальные окна и боковые панели (Drawers), чтобы мастер не терял контекст текущего дня.

### Основные компоненты:
1. **Header (Шапка)**: Навигация по датам и глобальные действия.
2. **Main Workspace (Рабочая область)**: Канбан/Календарная сетка постов.
3. **Side Drawer (Боковая шторка)**: Детальная информация о записи и быстрое редактирование.
4. **Modal Window (Модальное окно)**: Форма создания новой записи.

---

## 2. Структура Главного экрана (Календарь постов)

Оптимальный паттерн для автосервиса — **Вертикальный календарь (Timeline) с колонками для каждого поста**. Это позволяет мгновенно оценивать загрузку параллельных рабочих мест.

### Раскладка (Desktop):
```text
[ ЛОГО ] [ < Сегодня, 15 Окт > 📅 ]   [ Легенда ]  [ Свободных слотов: 4 ]  [ + НОВАЯ ЗАПИСЬ ]
-----------------------------------------------------------------------------------------
Время    | Пост 1 (Подъемник)       | Пост 2 (Шиномонтаж)      | Пост 3 (Диагностика)
-----------------------------------------------------------------------------------------
09:00    | [ Замена масла       ]   | [ Смена резины       ]   |
09:30    | [ Азиз (Telegram) ⏳ ]   | [ Рустам (С улицы) ✓ ]   | [ + ] (Свободное окно)
10:00    | ----------------------   |                          | [ Диагностика ДВС  ]
10:30    | [ Диагностика ходов..]   | [ Балансировка       ]   | [ Тимур (Telegram) ]
11:00    | [ ~ плавающее время ~]   | [ Ожидает запчасти ⏸ ]   | [ ~ плавающее ~~~~ ]
...
```

### Элементы управления:
*   **Таймлайн (ось Y)**: Шаг 30 минут. Текущее время отображается яркой горизонтальной красной линией ("Time indicator"), пересекающей все колонки, чтобы мастер сразу видел, кто сейчас должен быть на посту.
*   **Колонки (ось X)**: Физические посты. Если их много (больше 4-5), добавляется горизонтальный скролл.
*   **Пустые слоты**: При наведении курсора на пустую ячейку появляется полупрозрачная кнопка `[ + Добавить ]`. Клик сразу открывает форму записи на это время и этот пост.

## 3. Визуальная система и статусы (Карточка записи)

Мастер должен считывать суть за 1 секунду. Приоритет данных в карточке:
1. Статус (Цвет)
2. Время (Позиция и текст)
3. Автомобиль/Имя (Кто это)
4. Услуга (Что делать)

### Типографика и индикаторы внутри карточки:
*   **Заголовок (Bold)**: Имя клиента + Марка/Госномер (если есть).
*   **Подзаголовок (Regular)**: Название услуги (н-р, "Замена колодок").
*   **Иконка источника**: В правом верхнем углу (✈️ для Telegram-бота, 🚶‍♂️ для ручной записи "с улицы").
*   **Тайминг**: Слева или внизу (н-р, "14:00 - 15:30").

### Индикация типов услуг:
*   **Фиксированная услуга**: Карточка имеет сплошной цвет фона, чёткие границы.
*   **Плавающая услуга**: Нижняя граница карточки имеет "рваный" край или штрихованный паттерн (зигзаг/полоски), показывая, что окончание ориентировочное. Добавляется иконка ⏳.

### Цветовая система статусов:
| Статус | Внутреннее имя | Визуальное оформление |
| :--- | :--- | :--- |
| **Записан** | `scheduled` | Светло-серый фон, тёмно-серый текст. Нейтрально. |
| **Подтвердил** | `confirmed` | Светло-синий фон, синий текст/бордер. Ожидаем клиента. |
| **В работе** | `in_progress` | Светло-жёлтый/Оранжевый фон. Яркий левый бордер. Опционально: пульсирующая точка (🔴), привлекает внимание. |
| **Готово** | `done` | Светло-зелёный фон. Карточка становится визуально более "тусклой" (opacity 70%), так как задача завершена. |
| **Не приехал** | `no_show` | Бледно-красный фон. Иконка ⚠️. |
| **Отменён** | `cancelled` | Прозрачный фон с пунктирной рамкой, зачёркнутый текст. Занимает место, но визуально не мешает. |

## 4. Поведение и Взаимодействие (User Flow)

### Сценарий 1: Смена статуса записи (Самое частое действие)
1. Мастер кликает на карточку записи.
2. Справа въезжает **Side Drawer** (Боковая панель, перекрывает 30% экрана, не закрывая расписание полностью).
3. В шапке панели — текущий статус и крупные кнопки перевода в следующий логичный статус:
    *   Если статус `confirmed` -> Кнопка **[ Начать работу ]** (переведёт в in_progress) и **[ Не приехал ]**.
    *   Если статус `in_progress` -> Кнопки **[ Завершить ]** (done) и **[ Продлить время ]** (добавит +30 мин к слоту).
4. Ниже в панели: телефон (кликабельно для звонка), детали авто, история изменений статусов.

### Сценарий 2: Добавление записи с улицы
1. Мастер нажимает кнопку "+ Новая запись" в шапке ИЛИ кликает на пустой слот (например, Пост 2, 11:30).
2. Появляется **Модальное окно** (с фокусом по центру экрана).
3. **Шаг 1: Параметры времени**. Пост и Время уже заполнены (если клик из сетки).
4. **Шаг 2: Услуга**. Выпадающий список с автопоиском. При выборе услуги автоматически подставляется её стандартная длительность. Можно выбрать чекбокс "Плавающее время".
5. **Шаг 3: Клиент**. Поле "Имя" и "Телефон" (с маской `+998`).
6. Нажатие **[ Записать ]**. Модалка закрывается, на таймлайне появляется новая синяя карточка с иконкой 🚶‍♂️.

## 5. Пустые состояния (Empty States) и Ошибки

*   **На день нет записей**: Вместо сетки постов выводится приятная иллюстрация (чистый гараж) и текст: "На [Дата] записей нет. Свободных слотов: 24. Ждём клиентов из Telegram или с улицы!".
*   **Состояние загрузки**: При смене даты показывается "скелетный" экран (Skeleton loader) — серые мерцающие блоки на местах карточек, чтобы избежать резких скачков верстки.
*   **Офлайн / Ошибка сети**: Появление красного тоста сверху: "Нет подключения. Вы переведены в офлайн-режим. Пожалуйста, не закрывайте вкладку, данные синхронизируются при появлении сети".

## 6. Мобильная адаптация (Mobile View)
Мастер иногда отходит от компьютера и смотрит расписание со смартфона. Вертикальные колонки постов на узком экране не поместятся.

**Решение для мобильных:**
*   Вместо показа всех постов одновременно, используется **Таб-бар (Вкладки)**: `[Пост 1] [Пост 2] [Пост 3]`.
*   Мастер видит вертикальный таймлайн только для выбранного поста.
*   Свайп влево-вправо переключает посты.
*   Над табами показывается индикатор занятости (н-р, красная точка у Поста 2 означает, что там сейчас идёт работа).
*   Кнопка "+ Добавить" становится плавающей круглой кнопкой в правом нижнем углу (FAB).

## 7. Чек-лист приоритетов для UI-разработчика
1. **Плотность**: Интерфейс должен вмещать максимум информации без прокрутки (компактные отступы, мелкий, но читаемый шрифт для деталей).
2. **Контрастность статусов**: Мастер боковым зрением должен видеть жёлтые пятна (в работе) и красные (проблема/не приехал).
3. **Никаких подтверждений для безопасных действий**: Смена статуса происходит в 1-2 клика. Удаление или отмена записи — с запросом подтверждения.
4. **Текущее время**: Красная линия "сейчас" — самый важный ориентир, она должна всегда быть на виду (автоскролл к линии при открытии приложения).
